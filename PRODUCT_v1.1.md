## 1. 产品是什么

**AI Hedge Fund 中国版是一个本地运行的 A 股 / 港股投研可视化工作台 + 研究回路**：把分散在 baostock / Tushare / 腾讯·新浪 / 东方财富 / openclaw 的多维数据，在四个页面上同屏展示，辅助投资决策；并以每周批跑 + outcome 评估构成信号研究回路（故事 H）。

**定位的关键澄清**：

| 项 | openclaw 负责 | 本工具负责 |
|---|------------|----------|
| 早晚报 / 周复盘生成 | ✅ | ❌（消费 + 存档） |
| 联网搜索 / 跨平台舆情 | ✅ | ❌（消费 JSON 输出） |
| 港股新闻深度采集 | ✅ | ❌（消费 JSON 输出） |
| 个股多维数据可视化 | ❌ | ✅ |
| 多 agent 并行决议 | ❌ | ✅ |
| 三分法池管理与迁移 | ❌ | ✅ |
| 行情 / 财报数据（baostock 主源 + 数据源路由层，TECH 附录 A） | ❌ | ✅ |
| 批量信号生成与 outcome 评估（研究回路，D5） | ❌ | ✅ |

不是 SaaS、不是交易系统、不是报告生成器，是**本地的、按需的、面板式的投研辅助**。

## 3. 用户故事

### 故事 A — 粘贴 openclaw 早晚报 + 标的抽取（每日例行）

> 早上 openclaw 给我生成了早报，我打开本工具的「报告粘贴入口」页，整段粘贴进去：
>
> 1. 工具自动按日期 + 类型存档（`briefings_archive/2026-06-13-morning.md`）
> 2. **LLM 抽取早报中提到的所有标的**，列出可点击的卡片（含 ticker / 名称 / 在早报中的角色：重点关注 / 风险标的 / 一笔带过）
> 3. 每个卡片右侧"深度看看"按钮 → 跳到该标的的个股深度页
> 4. LLM 抽取失败时降级到"手动多选你池里的票"
> 5. 已存档的早晚报永久可在「历史浏览」页里全文搜索

晚报 / 周复盘同流程。这是工具的入口动作。

### 故事 B — 个股深度分析（核心场景，使用率最高）

> 看一只票（来自早报标的卡片、来自三分法池、或随机输入 ticker）：
>
> **一屏展示 10 个维度**：
>
> 1. **顶部 strip**：ticker / 名称 / 价格 / 涨跌 / 三分法状态徽章 / 市值
> 2. **K 线区**：日 / 周 / 月切换 + MA5/20/60 + 成交量副图
> 3. **核心指标 8 卡**：PE / PB / ROE / 股息率 / 营收 YoY / 净利 YoY / 负债率 / 机构持仓
> 4. **多 agent 决议（9-agent 推荐集，D8）**：估值 / 基本面 / Buffett / 技术 + 中国舆情 / 政策 / 资金流 / 板块轮动 + Taleb（风险叠加）→ 综合裁决；港股集以 Peter Lynch 替换部分 V 类视角（花名册与注册表见 TECH §6.4）
> 5. **DCF 估值卡**：可调假设（永续 g / WACC / 5 年增速），给出内在价值区间 vs 现价
> 6. **同业对比表**：申万二级行业，标的高亮，行业中位作底线
> 7. **同业指数叠加图**：标的 K 线 vs 同业指数（如申万白酒）
> 8. **资金面 sparkline**：北向持股 / 融资余额 / 主力资金 20 日趋势
> 9. **限售解禁雷达**：未来 3 / 6 / 12 个月解禁规模可视化（占流通比 + 历史解禁后 30 日股价表现）
> 10. **公告 + 新闻 + 研报时间线**：A 股从 AKShare 自动拉；港股从 openclaw JSON 粘贴

附加：**财务舞弊检测 agent**（独立卡片）—— 检查应收账款 / 存货 / 经营现金流 vs 净利匹配度等异常指标，给"健康 / 关注 / 警示"三档评分 + 具体异常项列表。

### 故事 C — 三分法池总览与操作（每周用 1-2 次）

> 在「三分法池」页：
>
> 1. **15 只一屏热力图**：V5 / T5 / N5 三列，每只票一个 cell，颜色编码 5 日 / 20 日涨跌
> 2. **迁移信号红黄灯**：每只票旁边显示当前是否触发迁移信号（T→V / V→T / N→T），点击查看具体证据
> 3. **手动操作**：增 / 减 / 迁移按钮（所有决策都是人工触发，工具不自动改池）
> 4. **月度配额显示**：本月已用 0/2 进 + 0/2 出，超额操作直接拒绝
> 5. **池累计回报**：每只票从入池起的累计表现
> 6. **自选股 15 只**：弹性池，与三大池区分

### 故事 D — 历史早晚报浏览（按需查询）

> 在「历史报告」页：
>
> 1. 按日期 / 类型（morning / evening / weekly）筛选
> 2. 全文搜索（grep over `_all_briefings.md`）
> 3. 选中一份报告 → 渲染 + 列出当时报告中提到的所有标的（用历史抽取结果，不重新抽）
> 4. 任何标的可一键跳到当前的个股深度页（注意：数据是当前的，不是历史的，便于"回看当时怎么想 vs 现在如何"对照）

### 故事 E — openclaw 港股新闻消费

> 看一只港股票（如 N1 优必选 09880.HK）的深度页，发现「公告 + 新闻 + 研报」时间线一片空白（A 股的 AKShare 接口不覆盖港股）。
>
> 操作：
>
> 1. 切到 openclaw，找到"个股新闻结构化输出"prompt，输入 ticker，跑一次
> 2. 复制 openclaw 输出的 JSON 块
> 3. 回到深度页，点右上角「导入 openclaw 新闻」按钮，粘贴
> 4. 工具按 JSON schema v1.0 校验：通过 → 自动填充时间线 + 风险事件 + 同业事件；失败 → 红底报错告诉哪个字段不对，让我重跑
> 5. 持久化到 `~/.ai-hedge-fund/hk_news/{ticker}_{YYYYMMDD}.json`，24 小时内打开同一只票自动加载，不要求重跑

### 故事 F — Multi-Provider 灵活切换

> 6 家 LLM API（DeepSeek 默认 + Claude / OpenAI / Gemini / Qwen / Kimi）：
>
> - `.env` 里有哪个就能用哪个
> - Web 控制台下拉切换 provider，不需要重启
> - DeepSeek thinking 模式（默认开启会让 JSON 任务返回空 content）必须强制 `"thinking": {"type": "disabled"}`

### 故事 G — 网络环境鲁棒性（同 v0.1）

> Windows 系统级代理（Clash / v2rayN TUN 模式）必须通过 NO_PROXY 进程级注入隔离 AKShare 数据源。详见 TECH.md。

### 故事 H — 批跑信号 + outcome 评估（研究回路，每周例行，D5）

> 每周让同事对 15 只池批跑一次（A 股 13 只与港股 2 只分开跑，市场模式自动切换）：
>
> 1. `main_china.py` 产出每只票的 agent 决议与 PORTFOLIO SUMMARY（action / confidence / 票数）
> 2. run log 直接喂 `src/eval/harness`，按信号日截面累积；前瞻收益到期后自动可算
> 3. 评估以**已实现收益为真值**（RankIC / 命中率 / 超额命中 vs 沪深 300），意见三角（Claude / openclaw）仅辅助（I10.5）
> 4. 回填历史日期评估前，必须先过 PIT 探针（I10.4）
> 5. **已知信号边界（2026-07 三截面初评）**：短端（5 日）排序弱有效（RankIC 三独立截面全正），方向命中对基准 ≈ 抛硬币；不许诺方向预测力，不据以自动交易

## 5. 产品不变量（Invariants — SDD 核心）

> **41 条不变量**：v1.0 实数 35 条（原文"32 条"/"23 条"及增删算术为计数错误，本版勘误）+ v1.1 新增 I10 域 6 条（研究回路与市场正确性，源自 2026-06 数据层 / 信号层实战教训）。

### I1 — 数据真实性

- **I1.1 缺数据必须显式标注**：报告或页面中任何数据点拿不到，必须标【数据缺口】，禁止默认为 0 / 空数组 / 编造。**已知反例（v1.1 对账实锤，修复列 backlog 首位）**：capflow 在无北向数据时输出 82% 高置信信号；technical 以窗口退化值 0 冒充 momentum_3m/6m、hurst 等真值。
- **I1.2 数据源 fallback chain 必须可追溯**：核心数据有 ≥ 2 个数据源备选，实际使用的链路在页面 footer 标注。
- **I1.3 港股数据弱默认值规则**：港股标的的 `china_public_opinion` agent 在无 openclaw 新闻输入时，必须返回中性信号 + 显式标注"无 openclaw 数据，建议导入"，禁止编造港股新闻。
- **I1.4 DCF 假设必须用户可见可调**：永续 g / WACC / 5 年增速三个核心假设在 UI 上必须显示当前值 + 滑块；硬编码默认值视为违反不变量。
- **I1.5 财务舞弊检测必须给具体指标，不能只给结论**：警示档必须列出具体异常项（如"应收账款增速 45% > 营收增速 12%"），不能只是一句"财务异常"。

### I2 — 早晚报消费完整性

- **I2.1 粘贴的早晚报必须正确归档**（按日期 + 类型 + .bak 滚动），内容不修改不重新生成。
- **I2.2 标的抽取 LLM 调用失败时降级到手动多选**：fail-loud 弹错误 + 显式标注"AI 抽取失败，请手动选择"，禁止静默跳过。
- **I2.3 标的抽取必须区分角色**：重点关注 / 风险标的 / 一笔带过 三类。
- **I2.4 已存档报告不可修改**：仅可重新粘贴生成 .bak.N，避免历史污染。

### I3 — 港股新闻 openclaw 输入消费

- **I3.1 openclaw 输入必须按 schema v1.0 校验**：缺字段降级（不报错但 footer 标注"数据不完整"）；字段类型错误整体拒绝 + 红底提示。
- **I3.2 港股新闻持久化路径**：`~/.ai-hedge-fund/hk_news/{ticker}_{YYYYMMDD}.json`，24 小时内自动加载。
- **I3.3 港股新闻消费后必须立即驱动多 agent**：`china_public_opinion` agent 必须读取这份 JSON 而不是返回中性。
- **I3.4 schema 版本前向兼容**：用 `schema_version` 字段判定解析路径，未来 v1.1 不能破坏 v1.0 的存档。

### I4 — 三分法约束（人工决策版）

- **I4.1 池规模锁定 V5 + T5 + N5**：任意时刻不能超员。
- **I4.2 月度迁移上限 2 进 2 出**：跨月清零，超额操作直接拒绝。
- **I4.3 迁移操作只能从总览页手动触发**：工具**不再**自动生成迁移建议（晚报已 outsource 给 openclaw），人工看到迁移信号红黄灯后决定是否操作。
- **I4.4 池状态持久化健壮**：`.tmp` 文件 + rename + 7 天滚动 backup。
- **I4.5 迁移记录含完整 evidence 字段**：人工触发时必须填理由（不能跳过），含具体数据点引用。

### I5 — 代码版本一致性（防 Ghost Version，同 v0.1）

- **I5.1 启动横幅必须打印关键模块的实际文件路径 + `__version__`**。
- **I5.2 每个核心模块有 `__version__` 字符串**。
- **I5.3 `tools.api_bridge` 必须在 agent 导入前 import**：违反此条 patch 失效但程序不报错。

### I6 — 容错与可恢复性

- **I6.1 LLM 调用失败必须 fail-loud**：禁止静默回退（标的抽取 / 多 agent / 舞弊检测三种调用都适用）。
- **I6.2 单标的数据失败降级**：标【数据缺口】，其他标的继续；≥ 50% 失败暂停整页。
- **I6.3 Prompt token 上限保护**：超限抛 `LLMPromptTooLongError`，不发请求。
- **I6.4 NO_PROXY 进程级注入**：必须在 `inject_no_proxy()` 之后才允许 import 任何调 AKShare 的模块。

### I7 — 用户控制与隐私

- **I7.1 API Key 在 `.env`**，不上传 / 不硬编码 / 不入 git。
- **I7.2 个人持仓 / 池配置本地存储**。
- **I7.3 Provider 切换在 Web 控制台即时生效**，不需要重启。

### I8 — 透明度

- **I8.1 每次 LLM 调用打印 (model, endpoint, prompt_tokens, completion_tokens, estimated_cost_usd)** 到日志和 Web UI。
- **I8.2 数据点可追溯到数据源**：footer 集中标注。
- **I8.3 页面 footer 必须包含**：数据时点 + 数据源链路 + 数据缺口列表 + LLM 调用元信息。

### I9 — 输出文件管理

- **I9.1 早晚报归档命名**：`~/.ai-hedge-fund/briefings_archive/{YYYY-MM-DD}-{morning|evening|weekly}.md`。
- **I9.2 重复粘贴保留 `.bak.N`**，不直接覆盖。
- **I9.3 港股新闻 JSON 归档**：`~/.ai-hedge-fund/hk_news/{ticker}_{YYYYMMDD}.json`。
- **I9.4 池状态备份滚动**：保留 7 天。

### I10 — 研究回路与市场正确性（v1.1 新增，实战伤疤规格化）

- **I10.1 TTM 口径强制**：流量科目（净利 / 营收 / FCF / capex / 折旧等）的消费必须为 TTM 口径；季报 YTD 累计值禁止直接当期使用。canonical 实现：`_ttm_from_ytd` 三条 overlay 规则（修①）。
- **I10.2 市场参数路由**：CN / HK 代码路径禁止美股参数硬编码；资本成本、基准指数、交易日历必须经 `markets/market_config.py` 按市场路由（单一真相源）。**任何模块自带绝对 WACC 表视为违反**（D4 v2）。
- **I10.3 无数据禁高置信**：agent 在关键输入缺失时禁止输出高置信方向信号，必须透传 `data_quality` 标志供下游降权与页面标注（capflow 82% artifact 教训；I6.1 的语义级补丁）。
- **I10.4 PIT 纪律**：任何带 end-date 的运行，全部数据读取不得越过 end-date；回填式评估执行前必须通过 PIT 探针（价格敏感 agent 双日期比对法，TECH §14 R9）。
- **I10.5 outcome 为真值**：信号质量结论必须以已实现收益为真值（`src/eval/` harness：RankIC / 命中率 / 超额命中）；意见三角仅作辅助参考，不得单独作为信号质量证据。
- **I10.6 单源熔断韧性**：任一单一数据源熔断，完整 run 不得中断（允许置信度下降与【数据缺口】标注，不允许崩溃）——DL-1 已真机验收。
